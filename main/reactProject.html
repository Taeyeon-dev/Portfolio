React project

React.js Calendar project




What it is

Using fullcalendar that is a JavaScript library that seamlessly integrates with such popular JavaScript frameworks

It is a calendar-schedular. User is available to update schedule and delete it. When finish the schedule, schedule color is changed green to red.


What I did

1. Use functional components only
Only functional components were used because the functional type was simpler and there was no need for a class type when using the react hook.

2. React hook (useState(), useDispatch(), useEffect(), useRef(), useSelect()) - When the Add button is pressed, the new schedule is saved in Firestore and redux,
and it goes to the main page and dispatches the middleware function that loads the data using useEffect.
Then use Select to retrieve the values stored in the redox and display them on the calendar page.
3. UI used css that can give elements detailed effects and styled-components for overall definition.

4. Data was given as a child component using State and Props.

5. Paging function was applied to components using Route.

(Error occurred when attempting to bind with Routes.)

6. Information was saved and retrieved using Redux.
-In the modal window, there are the delete schedule button and the complete button, and when you press the delete button,
the corresponding data in the firebase and readerx are found as id values and deleted.
The complete button finds the data with the id value and updates the data key value completed to true.

7. Data can be stored using Firebase (serverless).

8. I used middleware to connect Firebase to Redux.



Responsive Web

Web pages used media queries for use on PCs, tablets, and mobile phones.
When the width became smaller at 50vw, the width changed the value from 80vw to 100vw.
And when it was too big compared to the text screen, 'display: none' was used when the screen became smaller.
When the screen becomes smaller, the modal window button is hidden in the menu icon and set to active only when clicked.
